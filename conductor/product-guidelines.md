# Product Guidelines

## Communication Style
- **Technical and Precise:** The application uses technical language suitable for experienced developers and DevOps students. Accuracy and clarity are prioritized over conversational tone.

## Visual Identity
- **Dark/Terminal Aesthetic:** The UI features a developer-centric aesthetic with deep grays and blues, using vibrant accent colors to highlight important actions or status changes.

## Terminology
- **Industry Standard Jargon:** Docker-specific terminology (e.g., Image, Tag, Layer, Registry) is used directly and correctly throughout the interface to maintain consistency with industry tools.

## User Experience (UX)
- **Visual Clarity:** The layout prioritizes an uncluttered experience with a clear information hierarchy, ensuring that key image details are easy to find and digest.

## Error Handling and Feedback
- **Developer-Focused Details:** Error messages provide specific technical details and codes to assist users in troubleshooting network or API-related issues.
